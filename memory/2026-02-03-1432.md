### Skill 标准核心要素

```yaml
# Skill 定义示例结构

  name: "代码审查"
  description: "分析代码质量、潜在 bug 与安全漏洞"
  parameters:
    - name: "code"
      type: "string"
      description: "待审查的代码"
    - name: "language"
      type: "string"
      enum: ["python", "javascript", "go"]
  output_format: "markdown"
  capabilities:
    - "静态分析"
```

## 建议简化

| 接口 | 问题 | 建议 |
|------|------|------|
| `list_apis` | Agent 不需要"浏览"所有 API | ❌ 删除 |
| `search_apis` | Agent 知道要什么 API，无需搜索 | ❌ 删除 |
| `get_apis` | 了解接口定义有用 | ✅ 保留，改名 `get_api_info` |
| `call_apis` | 核心功能 | ✅ 保留 |

## 推荐接口设计

```typescript
// 核心接口
interface MCPHandlers {
  // 调用 API（最常用）
  call_api: {
    input: {
      api: string;        // 如 "openrouter", "ave", "twitter", "serper"
      method: string;     // GET/POST
      path: string;       // 具体 endpoint
      params?: object;    // 请求参数
      body?: object;      // 请求体
    };
    output: {
      success: boolean;
      data: any;
      error?: string;
    };
  };
  
  // 获取接口文档（可选，Agent 需要了解参数时用）
  get_api_info: {
    input: {
      api: string;
      path?: string;      // 可选，查看具体 endpoint
    };
    output: {
      description: string;
      parameters: object;
      examples: object[];
    };
  };
}
```

## 理由

1. **Agent 是目标驱动**：不是来"逛街"的，不需要 list/search
2. **最小化交互**：Agent 直接调用，不需要先查文档（除非遇到错误）
3. **按需获取**：`get_api_info` 仅在 Agent 不确定参数时调用

## 进阶（可选）

```typescript
// 复杂场景可选
batch_call: {
  // 批量调用，适合多平台聚合查询
  input: { requests: Array<{api, method, path, params}> };
  output: { results: Array<{api, success, data}> };
};
```

这样 Agent 只需要记一个 `call_api` 就能搞定所有集成场景。
